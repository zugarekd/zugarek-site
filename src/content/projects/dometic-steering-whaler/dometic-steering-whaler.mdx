## Overview

This project documents the bench setup and commissioning process for a **Boston Whaler 130 Super Sport** equipped with:

- **Simrad GO7**
- **Dometic Optimus EPS steering**
- **EA1000 autopilot module**
- **Precision-9 heading sensor**
- Full **NMEA 2000 backbone**

The goal was to create a stable, predictable network environment before launching ‚Äî including simulating engine RPM and water speed to validate autopilot and steering behavior.

---

## üéØ Goal

Establish a **stable speed and RPM source** on the NMEA 2000 network so that:

- The autopilot behaves correctly
- The helm display shows consistent data
- Steering calibration can be completed
- Limp-mode or fault conditions caused by missing data are eliminated

---

## Network Architecture

A standard NMEA 2000 backbone was used with:

- Proper termination (120Œ© at both ends)
- Dedicated power tee
- Individual drop cables per device
- Verified backbone voltage and resistance

Treat backbone power and termination as **Layer 1** ‚Äî if this isn‚Äôt correct, nothing else will behave predictably.

## üîß What It Simulates

The ESP32 broadcasts:

- **PGN 127488** ‚Äî Engine Rapid Update (RPM)
- **PGN 128259** ‚Äî Speed Through Water (STW/SOW)

This allows:

- Steering stop calibration
- Autopilot idle detection validation
- RPM display verification
- Network troubleshooting without the engine running

# Troubleshooting Notes

## Symptom

- Limp mode during bench setup
- Steering system refusing calibration

## Root Cause

RPM data was required on the appropriate CAN channel (CAN 3) for the system to recognize the engine as ‚Äúrunning‚Äù at idle.

Without valid RPM input, the system treated the setup as incomplete.

## Fix

- Ensured correct RPM PGN broadcast
- Verified source address
- Confirmed reception on network
- Validated via display diagnostics

# Lessons Learned

- Always keep a known-good spare GPS puck for isolation testing
- Backbone power + termination = foundation of everything
- Know which device is authoritative for each PGN
- Simulating data can dramatically accelerate commissioning
- Bench validation prevents on-water surprises

---

## Closing Thoughts

Modern marine electronics are networked systems. Treat them like distributed software stacks ‚Äî isolate variables, validate inputs, and control your test environment.

The ESP32 simulator turned what could have been multiple launch/debug cycles into a controlled lab exercise.
